Inspiration

I chose to make a remote controlled car for a very simple reason. I like cars. More specifically, over the past couple of years I have become fascinated with formula 1, a fast paced racing championship. Although it may appear to be just some cars driving around in circles very quickly, there is an incredible amount of engineering that occurs behind the scenes. New ideas on aerodynamics, bio feuls, reusable energy, and safety are all being pioneered by skilled engineers. It is because of this that I have been drawn to engineering to begin with and it only seemed fitting to make my final project have something to do with F1. Although, my RC car can not go 250 mph, or looks identitical to an F1 car, it was still an incredibly challenging project for a newbie engineer like me. Below is a breakdown of the steps I took and the components I used to make my RC car.

Main Body

For the Main body of the car, I decided to make it out of laser cut balsa wood. Before making the design for the laser cutting, I already knew that I wanted to have a specific pattern of 3 mm holes, sperated 5 mm apart from center to center. This is because I already had screws, nuts, and joints that were meant for 3 mm holes. The L-brackets I am using to connect the walls to the floor are 3D printed but are not of my creation. This pattern just made the assembly of my car incredibly easy and efficient and also had the added bonus of making a cool pattern. Here is a Fusion 360 model of my pieces, a downloadable DXF file, and photos of the screws, nuts, and joints as well as the wood post-cutting and assembly.

Downloadable .dxf file

The design I went for is quite minimalist and is based around the idea of a soap box car. It has one longer floor section, two side flaps, and a roof that is 4 centimeters shorter than the base. I intentionally made the roof shorter because I wanted to leave some room for my hand to enter the box in case I needed to grab or adjust something inside. The dimensions are as follows:

  • Floor: 21 5mm x 90 mm
  • Sides: 215 mm base, 60 mm height1, 40 mm height2
  • Front: 84 mm x 40 mm
  • Roof: 170 mm x 84 mm
    • I made each part as a sepearate file/body then imported them into a single file for assembly. I used the move/copy tool, more specifically the point to point translation tool, to align them properly. One of the only challenges I faced while building this was attaching the roof. Since I have pretty big hands, it was difficult to screw in the roof into the L-bracket form the inside. If I were to make another version of this car, I would have one, made it a little bigger so it could have a more accesible inside, and two, use more L-brackets since some of the pieces are a little unstable. I sadly did not have that many plastic L-brackets lying around, so I ended up using only 6. Maybe next time I would use 8 or 10.

    Attaching the Wheels

    Once I had the main body fo the car complete, I had to attach the wheels. Although this might sound easy, it wasn't. While prototyping with cardboard, I found that zipties seemed to work the best when trying to hold down the two DC motors. However, when I tried to do the same thing with my lasercut boards, I realized that the holes were too small. Luckily I was able to find a small-width zip tie in my kit which was able to fit into the 3mm hole. For the other DC motor, I had to cut a large zip tie down the middle to halve it's width and make it fit.

    For the front two wheels, which are not contorlled by motors, I was able to use twine from my kit to secure them down. The wheels are LEGO wheels and are attached to LEGO squares with a round peg. The steps I took while tying the wheels down are displayed through my diagrm below. The final tie is also shown more clearly below.

    Circuit

    The Circuit that I used is fairly simple but looks quite complicated due to the large number of female to female cables. The microcontroller I am using is the Adafruit Huzzah ESP32 Feather because it allows me to use the wifi which is requiered for my project. The first thing I did once my controller was attached to the breadboard was to connect the L9110 motor driver. For power I used the microcontrollers USB source since it delivered more power than the 3.3 V port. I plugged the driver into ground and then A1-A into pin 4, A1-B to pin 5, B1-A to 12, and B1-B to 27. From there I also added three LED's: red, green, blue and orange to 13, 33, 32 and 15 in that order. These lights are to be indicator lights for when the car performs certain tasks such as left and right turns, reversing, and moving forward.

    The blue and orange LED's are ticking out the sides of the car because that is where they will be in the final model but the red and green LED's are floating near the back since they will be going through the roof and front of the car. The circuitry looks quite messy but I assure you it is not that bad. If I were to do this again, I would try and organize my lines better so it is easier to spot when there is a problem. Similarly, I would try and find smaller female to female connectors since these are quite long.

    The Code

    The code that I am using is a code that I have been playing with and adapting since the week 9: Networking/ IoT class. This code works with Google Firebase, a free database to host projects, to create a website that controls the car at the push of a button. The first part of the code introduces the Arduino IDE libraries required for this project. Then comes definitions to connect to my WiFi, so the board can communicate with the website followed by my variables being defined to their respective pins. The Secret key is blank so people dont hack my board as well as my WiFi. Next comes the void Setup() function. Here, I define my motors and LED's as OUTPUT and have them begin in the off position. I also include more WiFi code that was given to us when setting up the firebase project on google. Finally, we arrive at the main code. Within the void loop() function, I have 8 different functions that all correspod to different buttons found on my webpage. They are: Forward, Reverse, Left Turn, Right Turn, Forward Slalom, Reverse Slalom, Dance Party, and Off. They all have a similar structure which uses the digitalWrite() function to control the speed and direction of the motors. If I were to do this again I would have loved to be able to control the speed of the car using PWM, pulse width modulation, but I wasn't able to figure itout in time for the presentation.

    Webpage

    While I was working on my code, I also worked to make the corresponding webpage that would control the car wirelessly. As mentioned before, I used Google's Firebase system to do so. Using the steps given by Proffesor Melenbrink on the course website, under the Week 9 tab, I was able to set up my data base and control the car through there. However, controlling the car through the database is very finicky since it is very case sensitive, and inputting a uppercase letter as lowercase will not allow it work. Because of this, I decded to make a webpage for a better user interface. Luckily, google firebase can do this for us. Following their steps, I created a new html page in my Final Project folder. Using the code structure Proffessor Melenbrink gave us, I plugged in the firebase generated configuration code so it connected with my project. From there, I edited the webpage as I would any other, to customize it to fit my needs. I added 6 more buttons, totaling to 8, and arranged them in a manner that made sense to me. In order to connect the buttons to the firebase database, which would then tell the corresponding code to run, I used the document.getElementById and addEventListener. This reads any button press then tells the database to update, which then forces a code to run. Here is my html code. If I were to do this again I would have tried to use blue tooth to see if there was less of a delay between a button press and the car actually moving.
